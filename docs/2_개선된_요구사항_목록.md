# 개선된 요구사항 목록 (EnjoyTrip Web Backend PJT)

## 1. 요구사항 개선 방향

- 기존 명세(F101~F114)를 기준으로, 실제 구현 가능한 범위를 명확히 분리했다.
- 백엔드 API/DB 연동 여부를 요구사항에 명시하여 검증 가능성을 높였다.
- 각 요구사항에 `우선순위`, `구현 상태`, `검증 기준`을 추가했다.

## 3. 비기능 요구사항 (Non-Functional Requirements)

| ID | 분류 | 요구사항 | 우선순위 | 구현 상태 | 검증 기준 |
|---|---|---|---|---|---|
| NFR-101 | 인코딩 | 요청/응답 UTF-8 강제 처리 | 필수 | 구현완료 | 한글 입력/응답 깨짐 없음 |

| NFR-103 | 런타임 | Tomcat 11 + Jakarta Servlet 6.1 호환 | 필수 | 구현완료 | 톰캣 배포 후 서블릿 매핑 동작 |
| NFR-104 | 가용성 | DB 스키마 자동 생성으로 초기 설정 단순화 | 추가 | 구현완료 | 최초 실행 시 `members/auth_logs/notices` 생성 |
| NFR-105 | 안정성 | 뉴스 수집 실패 시 fallback 데이터 제공 | 추가 | 구현완료 | 외부 RSS 실패 시 기본 뉴스 반환 |
| NFR-106 | 진단성 | DB 에러 상세 정보(`sqlState`, `errorCode`) 노출 | 추가 | 구현완료 | DB 실패 응답에 상세 필드 포함 |

## 4. 데이터 요구사항

| 엔티티 | 주요 속성 | 설명 |
|---|---|---|
| members | user_id, name, email, password, created_at, updated_at | 회원 계정 데이터 |
| auth_logs | user_id, event_type, logged_at | 로그인/로그아웃 이벤트 이력 |
| notices | title, content, author, created_at, updated_at | 공지사항 게시글 |

## 5. API 요구사항

- 경로 최적화: `GET /api/route/optimize?points=...`
- 경로 일차 분할: `GET /api/route/split-by-day?points=...&days=...`
- 앱 상태: `GET /health`
- DB 상태: `GET /api/db/health`
- 회원: `GET/POST /api/members`
- 공지사항: `GET/POST /api/notices`
- 뉴스: `GET /api/news`

## 6. 향후 개선 항목

- 게시판(board) DB 연동 (현재 프론트 저장소 중심)
- 비밀번호 암호화(해시) 및 인증 토큰 기반 세션 고도화
- 관광/뉴스 데이터 캐시 및 서버 측 페이징
