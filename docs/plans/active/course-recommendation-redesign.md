# [Plan] 코스 추천 재설계 — 사용자 프로필 임베딩 + 재정렬

## 배경 및 목적

기존 추천 방식은 사용자가 저장한 장소/쪽지 임베딩을 평균 내어 선호 벡터를 만들고,
코스 아이템 임베딩 평균과 비교한다. 이 방식은 개별 아이템 벡터 노이즈가 심하고
코스 전체 성격을 반영하지 못한다.

**새 방식**:
- 사용자가 저장한 장소 + 쪽지를 LLM으로 프로필 설명으로 만들고 임베딩 → `member_profile_embeddings`
- 코스 설명을 LLM으로 생성하고 임베딩 → `course_embeddings` (API 응답 분리)
- 벡터 유사도로 후보 K개 조회 → 패널티 기반 재정렬

## Phase 3 — 회원 프로필 임베딩 (이벤트 기반)

### 3-A. 이벤트 기반 실시간 갱신 (primary)

장소/쪽지 저장·취소 시점에 프로필 임베딩을 즉시 재생성한다.
기존 `NoteEmbeddingRequestedEvent` → `NoteEmbeddingEventListener` 패턴과 동일하게 구성한다.

#### 이벤트 발행 (core-api domain)

`MemberProfileEmbeddingRefreshRequestedEvent(Long memberId)` 신설.

- `AttractionService.addSave()`: `attractionMapper.insertSave()` 성공 후 이벤트 발행
- `AttractionService.removeSave()`: `attractionMapper.deleteSave()` 성공 후 이벤트 발행
- `NoteService.addSave()`: `noteMapper.insertSave()` 성공 후 이벤트 발행
- `NoteService.removeSave()`: `noteMapper.deleteSave()` 성공 후 이벤트 발행

#### external 클라이언트 신설

`external` 모듈 — `com.ssafy.enjoytrip.external.profile` 패키지:

| 파일 | 역할 |
|------|------|
| `MemberProfileInput.java` | LLM에 전달할 저장 장소+쪽지 목록 DTO |
| `MemberProfileDescriptionResult.java` | LLM이 생성한 설명 + 임베딩 결과 |
| `MemberProfileEmbeddingException.java` | 실패 예외 |
| `MemberProfileEmbeddingClient.java` | ChatClient로 설명 생성 → EmbeddingModel로 임베딩 (두 단계 내부 처리) |

`MemberProfileInput`은 `external`이 core/domain 타입을 의존하지 않도록
저장 장소(이름·주소·카테고리)와 저장 쪽지(제목·카테고리·태그명) 목록을 plain 필드로 보유한다.

#### 프롬프트 전략

```
다음은 사용자가 저장한 장소와 쪽지 목록입니다.
이 정보를 바탕으로 사용자의 여행 취향을 2~3문장으로 설명해주세요.

[저장한 장소]
- (장소명, 주소, 카테고리) ...

[저장한 쪽지]
- (제목, 카테고리, 태그) ...
```

#### 이벤트 리스너 신설 (core-api)

`MemberProfileEmbeddingEventListener` — `@Async` + `@TransactionalEventListener(AFTER_COMMIT)`:

1. `attraction_saves` + `attractions`를 조인해 저장 장소 목록 조회 (새 mapper 메서드)
2. `note_saves` + `notes` + `note_tags` + `tags`를 조인해 저장 쪽지+태그 목록 조회
3. `MemberProfileInput` 조립 → `MemberProfileEmbeddingClient.embed()` 호출
4. 결과를 `member_profile_embeddings`에 upsert (새 mapper 메서드)

변경 감지: 장소+쪽지 목록 직렬화 SHA-256이 기존 `source_hash`와 같으면 임베딩 호출 건너뜀.

저장 장소/쪽지가 없는 사용자는 리스너에서 조기 반환 (빈 프로필 저장 불필요).

#### 신설 파일 (core-api)

```

  domain/event/
    MemberProfileEmbeddingRefreshRequestedEvent.java
  domain/event/listener/
    MemberProfileEmbeddingEventListener.java
```

```

    MemberProfileInput.java
```

## Phase 5 — 추천 로직 재작성 (storage:db-core + core-api)

### 5-1. 후보 조회 SQL (CourseMapper.xml)

기존 `findRecommendedCourses`를 대체. 반환 레코드는 `CourseRecommendationCandidateRecord`(신설)로
`dominant_category`, `similarity_distance` 포함.

```sql
-- findCandidatesByMemberProfile(memberId, candidateLimit)
WITH member_vec AS (
    SELECT embedding
    FROM member_profile_embeddings
    WHERE member_id = #{memberId}
      AND status = 'EMBEDDED'
      AND embedding IS NOT NULL
)
SELECT c.*, ce.dominant_category,
       ce.embedding <=> mv.embedding AS similarity_distance
FROM courses c
JOIN course_embeddings ce ON ce.course_id = c.id
CROSS JOIN member_vec mv
WHERE c.deleted_at IS NULL
  AND c.visibility = 'PUBLIC'
  AND ce.status = 'EMBEDDED'
  AND ce.embedding IS NOT NULL
ORDER BY similarity_distance ASC
LIMIT #{candidateLimit}
```

추가 mapper 메서드:
- `hasMemberProfileEmbedding(Long memberId) → boolean`
- `findRecentlyViewedCourseIds(Long memberId, int days) → List<String>`

### 5-2. 재정렬 서비스 (core-api domain)

```java
// com.ssafy.enjoytrip.core.domain.CourseRecommendationRanker
public List<Course> rerank(
    List<CourseRecommendationCandidate> candidates,
    RerankingContext context,
    int limit
)
```

**`RerankingContext`**:
- `Set<String> recentlyViewedCourseIds` — 최근 30일 조회 이력
- `Map<Long, Long> memberTagFrequency` — note_tags에서 사용자 태그별 빈도

**패널티 규칙** (초기 점수 = `similarity_distance`, 낮을수록 좋음):

| 조건 | 패널티 |
|------|--------|
| 최근 7일 내 본 코스 | +0.30 |
| 최근 8~30일 내 본 코스 | +0.15 |
| 선택된 후보 중 같은 `dominant_category`가 이미 2개 이상 | +0.20 × 초과 수 |
| 코스 태그가 사용자 상위 3개 태그에만 집중 (편향) | +0.10 |

재정렬 방식: greedy 소비 — 후보를 점수 오름차순으로 순회하며 카테고리 쿼터를 추적해 선택.

### 5-3. CourseService 수정

```java
public List<Course> findRecommendations(Long memberId, String regionName, int limit) {
    if (!courseReader.hasMemberProfileEmbedding(memberId)) {
        return courseReader.findPopularByRegion(regionName, limit);
    }

    List<CourseRecommendationCandidate> candidates =
        courseReader.findCandidatesByMemberProfile(memberId, limit * 3);

    RerankingContext context = buildRerankingContext(memberId);

    return ranker.rerank(candidates, context, limit);
}

private RerankingContext buildRerankingContext(Long memberId) {
    Set<String> recentViews = courseReader.findRecentlyViewedCourseIds(memberId, 30);
    Map<Long, Long> tagFrequency = noteTagReader.findMemberTagFrequency(memberId);
    return new RerankingContext(recentViews, tagFrequency);
}
```
