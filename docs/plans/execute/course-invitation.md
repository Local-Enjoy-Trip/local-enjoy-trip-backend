# 코스 초대 기능 계획

## 브랜치

`feature/course-invitation`

---

## 기능 요약

- 코스 호스트(생성자)가 친구를 이메일로 초대 → 알림 발송
- 초대받은 친구가 수락하면 멤버가 되고 `GET /api/courses/me`에 해당 코스가 포함
- 호스트만 코스 수정/삭제 가능 (기존 `requireOwnedBy` 그대로 활용)

---

## Step 1: DB Migration

**신규 파일:** `storage/db-core/src/main/resources/db/migration/V25__add_course_invitations.sql`

```sql
CREATE TABLE course_invitations (
    id                BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    course_id         VARCHAR(128) NOT NULL,
    inviter_member_id BIGINT NOT NULL,
    invitee_member_id BIGINT NOT NULL,
    status            VARCHAR(20) NOT NULL DEFAULT 'PENDING',
    created_at        TIMESTAMP(6) NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at        TIMESTAMP(6) NOT NULL DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT fk_ci_course  FOREIGN KEY (course_id)         REFERENCES courses(id) ON DELETE CASCADE,
    CONSTRAINT fk_ci_inviter FOREIGN KEY (inviter_member_id) REFERENCES members(id) ON DELETE CASCADE,
    CONSTRAINT fk_ci_invitee FOREIGN KEY (invitee_member_id) REFERENCES members(id) ON DELETE CASCADE,
    CONSTRAINT uk_ci_course_invitee UNIQUE (course_id, invitee_member_id)
);
CREATE INDEX idx_ci_invitee_status ON course_invitations(invitee_member_id, status);
```

## Step 3: `storage/db-core`

### 신규

**`CourseInvitationRecord.java`**
- 필드: id, courseId, inviterMemberId, inviteeMemberId, status, createdAt, updatedAt

**`CourseInvitationMapper.java`**

| 메서드 | 설명 |
|--------|------|
| `insert(record)` | 초대 생성 |
| `findById(id)` | 단건 조회 |
| `findByCourseId(courseId)` | 코스별 초대 목록 |
| `updateStatus(id, status, updatedAt)` | 상태 변경 (수락/거절) |
| `existsByCourseAndInvitee(courseId, inviteeMemberId)` | 중복 초대 방지 |
| `findAcceptedCourseIdsByInvitee(inviteeMemberId)` | 수락한 코스 id 목록 |

**`CourseInvitationMapper.xml`** — 위 메서드 SQL 구현

### 수정

| 메서드 | 설명 |
|--------|------|
| `existsUnreadCourseInvitation(recipientMemberId)` | unread 초대 알림 존재 여부 |
| `findUnreadCourseInvitations(recipientMemberId, limit)` | 미읽은 초대 알림 목록 |

## Step 4: `core/core-api` — 도메인/서비스

### 신규

**`CourseInvitation.java`** — domain record
```
(Long id, String courseId, Long inviterMemberId, Long inviteeMemberId,
 CourseInvitationStatus status, LocalDateTime createdAt, LocalDateTime updatedAt)
```

**`event/CourseInvitationSentEvent.java`**
```
(Long invitationId, String courseId, String courseTitle, Long inviterMemberId, Long inviteeMemberId)
```

**`event/listener/CourseInvitationNotificationEventListener.java`**
- `@Async @TransactionalEventListener(phase = AFTER_COMMIT)`
- `NotificationService.saveCourseInvitationNotification(...)` 호출

**`service/CourseInvitationService.java`**

| 메서드 | 검증 |
|--------|------|
| `inviteFriend(hostMemberId, courseId, inviteeEmail)` | 코스 존재 + 호스트 확인, 친구 관계 확인(`FriendshipMapper`), 중복 초대 방지 |
| `acceptInvitation(inviteeMemberId, invitationId)` | invitee 일치 + PENDING 상태, `course_saves` 자동 저장 |
| `rejectInvitation(inviteeMemberId, invitationId)` | invitee 일치 + PENDING 상태 |
| `findByCourse(hostMemberId, courseId)` | 호스트 확인 |

### 수정

**`ErrorType.java`** — 4개 추가

| 코드 | HTTP | 메시지 |
|------|------|--------|
| `COURSE_INVITATION_NOT_FOUND (CI001)` | 404 | 코스 초대를 찾을 수 없습니다. |
| `COURSE_INVITATION_ACCESS_DENIED (CI002)` | 403 | 다른 사용자의 코스 초대에 접근할 수 없습니다. |
| `COURSE_INVITATION_ALREADY_EXISTS (CI003)` | 409 | 이미 초대가 존재합니다. |
| `NOT_FRIENDS (CI004)` | 400 | 친구 관계가 아닌 사용자는 초대할 수 없습니다. |

**`NotificationService.java`**
- `saveCourseInvitationNotification(invitationId, courseTitle, inviterMemberId, inviteeMemberId)` 추가
- `hasUnreadNotification`: friendship + course invitation 모두 체크
- `findNotifications`: friendship + course invitation 모두 반환

**`CourseReader.java`** — `CourseInvitationMapper` 주입 추가, `findMyCourses` 수정

```
findMyCourses(memberId):
  ① courseMapper.findByOwnerMemberId(memberId)                     → 내가 만든 코스
  ② invitationMapper.findAcceptedCourseIdsByInvitee(memberId)      → 수락한 코스 id 목록
  ③ 각 courseId → courseMapper.findById → findCourse(record, false) (private items 미포함)
  ④ ① + ③ 합쳐서 반환
```
